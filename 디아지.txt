93C46/93C56 메모리 보드 검사기 - 프로그램 작성 지시서
=======================================================

1. 하드웨어 구성
----------------

MCU: STM32H562RGT6 (LQFP-64), FreeRTOS 사용
프로젝트 경로: D:\stm32\93c56
기존 테스트 프로그램(LED+음성 동작 확인 완료)을 기반으로 작성한다.

[메인보드]
  - SK9822 LED x12 (SPI1: PA5=SCK, PA7=MOSI)
  - MAX98357A 음성 앰프 (I2S3: PA4=WS, PB2=SDO, PC10=CK)
    * SD_MODE = PB8 (R47 680k 풀업 → 항상 ON, GPIO로 제어하지 말 것)
  - CD4067 16ch 아날로그 멀티플렉서 (DO 라인 선택용)
  - 포그핀 커넥터 x12 (각 6핀: VCC, GND, CS, CLK, DI, DO)

[메모리보드]
  - 93C46 또는 93C56 EEPROM x12 (4열 x 3행)
  - 둘 다 x8 모드 (93C46: 128B/7bit주소, 93C56: 256B/8bit주소)
  - 포그핀으로 메인보드에 장착 (손으로 올림)

[GPIO 핀 매핑]
  93C56 Microwire:
    CLK  = PA8    (공유, 전 칩 공통)
    DIN  = PC0    (공유, MCU→칩)
    DOUT = PC1    (CD4067 출력→MCU)

  CD4067 MUX 선택:
    A=PC6, B=PC7, C=PC8, D=PC9, INHIBIT=PB12

  칩 셀렉트 (개별):
    CS0=PA9, CS1=PA10, CS2=PA11, CS3=PA12
    CS4=PB4, CS5=PB5, CS6=PB6, CS7=PB7, CS8=PB9
    CS9=PC11, CS10=PC12, CS11=PD2

[LED 물리적 배치와 체인 순서]
  보드 위 배치:              SK9822 체인 인덱스:
    U19 U20 U21 U22            0   1   2   3
    U23 U24 U25 U26            7   6   5   4   ← 중간행 역순
    U27 U28 U29 U30            8   9  10  11

  led_order[12] = {0,1,2,3, 7,6,5,4, 8,9,10,11}
  위치 인덱스 i → 체인 인덱스 led_order[i]

  주의: 위치4=U26(체인4), 위치7=U23(체인7) — 중간행이 뒤집혀있음


2. 동작 흐름
-------------

[전원 ON]
  (1) 음성: "이장치는 디에스이 아이엔시의 디아지 팁용 프로그래머입니다" (최초 1회)
  (2) 음성: "디아지용 팁 보드를 올려주세요"

[상태 A: LED 쇼 + 보드 대기]
  - U19→U30 순차 백색 점등 (200ms 간격)
  - LED 쇼 8패턴(A~H) 연속 반복, 매 패턴마다 색상 변경:
      A: 번호순 왕복     B: 세로 지그재그    C: 대각선 쓸기
      D: 소용돌이        E: 확산/수축        F: 랜덤 반짝
      G: 행/열 스캔      H: 체커보드
  - A~H 전체 완료 시 음성 반복: "디아지용 팁 보드를 올려주세요"
  - LED 쇼는 끊김 없이 진행 (보드 감지는 별도 태스크에서 수행)

[상태 B: 보드 감지 + 칩 식별]
  - 감지 확인 후 칩 종류 식별 (주소 200에 쓰기 시도 → 93C56 or 93C46)
  - 음성: "93C56 보드입니다" 또는 "93C46 보드입니다"
  - LED: 인식=백색, 미인식=적색

[상태 C: 메모리 검사]
  - 전 주소에 0xAA 쓰기→읽기 검증, 0x55 쓰기→읽기 검증
  - LED: 정상=백색, 불량=적색

[상태 D: 프로그래밍]
  - 전 주소에 0x00 쓰기 + 검증
  - LED: 완료=청색, 불량=적색
  - 음성: "프로그램이 완료되었습니다"
  - 불량 시: "적색 표시는 불량입니다"

[상태 E: 보드 제거 대기]
  - 음성: "보드를 분리하여 주십시요" (30초마다 반복)
  - 보드 분리 감지 시: "보드가 분리되었습니다"
  - LED 소등 → 상태 A로 복귀 (인트로 음성 없이 LED 쇼부터)


3. 안전 요구사항
-----------------

포그핀 접촉 불안정 대책:
  - 보드 감지 태스크(detectTask)는 읽기 전용만 수행 (쓰기 금지)
  - 3회 연속 동일 결과 확인 후에만 "감지됨"으로 판정
  - 쓰기 작업(검사/프로그래밍)은 안정 접촉 확인 후에만 수행

FreeRTOS 호환:
  - HAL_GetTick()이 FreeRTOS에서 증가하지 않을 수 있으므로
    ee_wait_ready() 등 타임아웃은 루프 카운터 방식으로 구현


4. 태스크 구조
---------------

  mainAppTask (Normal, 256x4):
    LED 쇼 + 상태머신(B~E) + 음성 재생 (I2S 직접 호출)

  detectTask (BelowNormal, 256x4):
    백그라운드 보드 감지, volatile 전역변수로 결과 전달
    g_board_present, g_detected[12], g_detect_enable

  defaultTask (Normal, 128x4):
    CubeMX 자동생성, 빈 루프

  주의: FreeRTOS 힙 = 8KB → 태스크 스택 합계 초과하지 않도록 관리


5. 음성 메시지 (8종)
---------------------

  #0 "이장치는 디에스이 아이엔시의 디아지 팁용 프로그래머입니다"
  #1 "디아지용 팁 보드를 올려주세요"
  #2 "프로그램이 완료되었습니다"
  #3 "적색 표시는 불량입니다"
  #4 "보드를 분리하여 주십시요"
  #5 "93C56 보드입니다"
  #6 "93C46 보드입니다"
  #7 "보드가 분리되었습니다"

  gen_voice.py로 Windows TTS(Heami) 자동 생성, 8kHz 16bit 모노 PCM
  음량 3배 증폭 적용


6. LED 등휘도 보정
-------------------

  인간 눈 감응도 (ITU-R BT.709) 기반으로 SK9822 brightness 레지스터를 색상별 개별 설정.
  적색 br=4, 청색 br=12, 녹색 br=4 등 → 체감 밝기 Y≈108 통일.
  백색 배경 LED: brightness=0 (최저, 색상 LED와 구분)
